### 检测请求信息

user-agent(ua)：用户代理

eferer：用来标识这个请求是从哪个网页过来的

cookie：：网站为了辨别用户进行会话跟踪而储存在用户本地的数据，主要功能是维持当前访问会话



添加请求头的时候：
​        如果给的请求头有错误，那么请求必定出错。
​        如果某个请求头没有，不一定出错。
同一个 ua【可以识别客户端信息的一个字符串】，如果使用同一个ua发送非常多请求，可能会封禁。
​    封禁会封多久？ 不知道。 封禁有网站自己设置。
​    为了防止这样的事情出现：
​        每次提交的时候，不使用相同的 ua

## URL

​    URL 叫统一资源定位符，对于一般的网页链接，习惯称为 URL
​    组成格式：
​        协议://主机地址[/路径][?查询]
​               域名【主机 电脑主机 =》电脑】
​               电脑的地址：电脑在哪里。
​               www.baidu.com => 153.3.238.127【ipv4地址 =》网站所在电脑的地址】

​            路径:
​                电脑里面的文件夹【电脑里面只有一个文件夹？】 =》相当于找电脑里面不同的文件夹

例如：
​    https://www.baidu.com/dps/?username=kspider&type=spider
​       协议：https
​       主机地址：www.baidu.com
​       路径：/dps/
​       查询：
​        username=kspider
​        type=spider

## 协议

​    约定，能干嘛不能干嘛

###   HTTP 

HTTP 超文本传输协议是一个简单的请求-响应协议，它指定了客户端可能发送给服 务器什么样的消息以及得到什么样的响应。
​        超文本：超越文本 =》图片、视频、音频【包含文本】
​        传输：通过网络传输，客户端【浏览器】和服务端【网站】
​        协议：约定浏览器和网站之间能传递什么，不能传递什么，按照什么格式传递。

###     HTTPS

​        S =》Secure =》锁 S
​        在 HTTP 的基础上加了一把锁，让数据更加安全。

## 查询字符串

查询字符串的缺点：
​    https://owssso.szse.cn/sso/login?用户名=13800138000&密码=123456

发送请求的时候，又两种参数：
    1. 查询字符串参数：可以直接在 url 通过 ？携带
        查询字符串参数，既可以出现在 get 方式，也可以出现在 post 方式
        2. 表单数据：在 url 后面看不到
        携带方式只有一种
        data 参数：
            ​    requests.get(url, data={"key":"value"})
            ​    参数形式：字典。
            ​    表单参数只会出现在 post 方式里面

批量修改：按 alt 键不放，点击鼠标左键

批量修改：

1.将复制过来的内容改成 key value 在同一行，中间要用东西隔开
accept: */*
accept-encoding: gzip, deflate, br, zstd

2.按 `ctrl + r`【替换】

3.复制任意一行到上面输入框

4.修改匹配规则【正则表达式】
一个小括号代表一个组
如果要用第几个组里面的内容，就用$序号
$1 $2
'$1': '$2',
'key': 'value',

`shift + 4 => 英文符号`

5. 点击最右边 的 “.*”
